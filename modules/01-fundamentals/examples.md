# 模块01 实践示例

## 示例 1：Temperature 参数对比实验

**任务**：为一款咖啡写一个产品介绍句（20字以内）

**temperature = 0（每次完全相同）**
```
输出：晨曦手冲咖啡，精选优质豆源，唤醒每个清晨。
（5次运行，结果100%一致）
```

**temperature = 0.7（有变化但合理）**
```
第1次：晨曦手冲，从第一滴开始，让早晨有了意义。
第2次：用一杯晨曦，开启你最好的一天。
第3次：晨曦手冲咖啡——慢生活，好味道。
```

**temperature = 1.2（高创意，偶有不连贯）**
```
第1次：每一滴晨曦，都是时间酿造的诗。
第2次：把山与云泡进杯里，这就是晨曦。
第3次：晨曦，不只是咖啡，是早晨送给自己的礼物。
```

**结论**：
- 事实性/一致性任务 → temperature=0
- 日常文案 → temperature=0.5~0.7
- 创意头脑风暴 → temperature=0.8~1.0

## 示例 2：系统提示 vs 用户提示的区别

**场景**：控制输出语言

**方案A：只在用户提示中说明（不稳定）**
```
用户：帮我介绍一下Python，用英文回答。
结果：Python is a high-level programming language...
（多数情况有效，但在长对话中可能被遗忘）
```

**方案B：在系统提示中锁定（稳定）**
```
系统提示：You must always respond in English,
          regardless of what language the user writes in.
用户：帮我介绍一下Python
结果：Python is a high-level, general-purpose programming language...
（始终稳定，无论对话多长）
```

**关键认知**：系统提示优先级更高，用于全局行为约束；用户提示用于具体任务。

## 示例 3：上下文位置的影响（"迷失在中间"效应）

**实验设置**：在3000字文本中藏入"重要联系人：张伟，电话：138-0000-0001"

```
位置A（开头第2段）→ 模型找到的概率：~95%
位置B（中间第15段）→ 模型找到的概率：~60%  ← 注意力下降！
位置C（结尾倒数第2段）→ 模型找到的概率：~90%
```

**实际应用**：在长提示词中，把最重要的指令放在开头或结尾，避免埋在中间。

## 示例 4：Token 数对成本的影响

**任务**：翻译同一段话，分析 Token 效率

**原版提示词（冗长版，约120 Token）**
```
我想请你帮助我把下面这段话翻译成英文，请你用专业、地道的英文表达，
确保翻译准确，同时保持原文的语气和风格，谢谢你的帮助：

[要翻译的内容]
```

**精简版提示词（紧凑版，约15 Token）**
```
将以下内容翻译成专业英文：

[要翻译的内容]
```

**结果**：两个版本翻译质量几乎相同，但精简版节省了 80% 的系统提示 Token。

**结论**：提示词不是越长越好，精炼表达更高效。

## 示例 5：LLM 数学能力的局限

**测试**：直接让 LLM 计算 `3721 × 4567`

```
不使用思维链（直接计算）：
提示词："3721 × 4567 = ?"
输出：16,994,207（错误！正确答案是 16,993,207）

使用计算工具提示（正确做法）：
提示词："使用Python计算 3721 × 4567，给出代码和结果"
输出：print(3721 * 4567)  # = 16,993,207  ← 正确
```

**关键认知**：LLM 不擅长精确数学计算（数字被逐 Token 处理，不是真正运算）。
生产环境中对数学计算应使用工具（计算器 API、代码执行）而非让 LLM 直接计算。
