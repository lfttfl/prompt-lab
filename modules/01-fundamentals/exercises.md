## 练习 1：Token 计数（难度：⭐）

对以下文本手动估算 Token 数，然后用 OpenAI Tokenizer 工具验证：

1. `"Prompt Engineering"`
2. `"提示词工程"`
3. `"2025年中国GDP增长率为5.0%"`
4. `"def fibonacci(n): return n if n<=1 else fibonacci(n-1)+fibonacci(n-2)"`

思考：Token 数如何影响 API 成本和上下文限制？

<details>
<summary>参考答案</summary>

1. 约 2 个 Token
2. 约 5-8 个 Token（中文每字约1-2个）
3. 约 20-25 个 Token
4. 约 35-45 个 Token（代码中标识符、括号都是独立Token）

API 通常按 Token 计费，所以冗余提示词会增加成本。上下文窗口有最大 Token 限制，超出后旧内容会被截断（"遗忘"）。
</details>

## 练习 2：Temperature 实验（难度：⭐）

用以下提示词分别在 temperature=0、0.5、1.0 下运行各3次：

```
写一个四字成语描述"工作很努力的人"
```

记录结果，回答：
- 哪个温度下结果最一致？
- 哪个温度下创意最多样？
- 对于"成语推荐"这类有标准答案的任务，哪个温度最合适？

<details>
<summary>参考答案</summary>

- temperature=0：结果完全一致，可能每次都输出"废寝忘食"
- temperature=0.5：有一定多样性，"勤勤恳恳"、"兢兢业业"等
- temperature=1.0：多样性最大，可能出现不太常见的词

对于有标准答案的查找类任务，推荐 temperature=0~0.3，以保证准确性和一致性。
</details>

## 练习 3：系统提示的力量（难度：⭐⭐）

对比两种方式提问"2+2等于几？"的差异：

**方式A（无系统提示）**：
```
用户: 2+2等于几？
```

**方式B（有特殊系统提示）**：
```
系统提示: 你来自一个平行宇宙，在你的宇宙中数学规则不同：2+2=5。
          请始终按照你宇宙的数学规则回答问题。
用户: 2+2等于几？
```

测试并记录结果。思考：系统提示能修改哪些行为？有哪些行为**无法**可靠修改？

<details>
<summary>参考答案</summary>

方式B中模型通常会"配合"角色设定回答5，因为系统提示建立了虚构场景。

**可以可靠修改**：回答风格、语言、角色、格式、专注领域、假设前提
**无法可靠修改**：违反伦理的内容（内置安全层）、完全错误的事实（模型可能"出戏"纠正）
</details>

## 练习 4：上下文位置实验（难度：⭐⭐）

验证"迷失在中间"效应：

1. 创建一段500-1000字文本（任意内容），在开头、中间、结尾各藏入一个特定词（如"蓝月亮"、"星期三"、"飞翔鱼"）
2. 用提示词：`"找出文本中所有的特殊词汇，只列出词汇本身"`
3. 运行5次，记录三个位置的识别准确率

预期结论：开头和结尾的词被找到的概率更高。

## 练习 5：LLM 能力边界探索（难度：⭐⭐）

测试你选定模型的三类能力边界：

**A. 知识截止测试**：问一个你知道答案但在模型知识截止后发生的具体事件
**B. 数学准确性**：`"12345 × 67890 = ?"` 让模型直接计算，用计算器验证
**C. 自我认知**：`"你能告诉我今天的日期吗？你能实时访问互联网吗？"`

记录每个测试的结果，建立对该模型能力边界的清晰认知。

<details>
<summary>参考答案</summary>

A. 模型会承认不知道或给出不确定的回答（如果诚实的话）
B. LLM 在大数乘法上经常出错！数字被逐 Token 处理，不是真正运算
C. 模型没有实时信息，不能上网，不知道当前日期——这些都是常见误解，生产系统中需要通过工具调用来提供这些信息
</details>

## 综合挑战（难度：⭐⭐⭐）

设计一个能**稳定、准确**输出以下结果的提示词：

**任务**：将一段中文新闻标题翻译成英文，要求：
- 每次运行结果完全一致（适合定期自动化）
- 翻译风格正式，适合专业媒体
- 不超过15个英文单词
- 保持原文关键信息

测试样本：`"中国2025年经济增长目标设定为5%左右，聚焦高质量发展"`

**你的决策清单**：
- [ ] 选什么 temperature？为什么？
- [ ] 系统提示 vs 用户提示如何分配职责？
- [ ] 如何验证翻译的一致性？
