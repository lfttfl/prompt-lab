# 模块 03：Few-shot 与示例驱动学习

> 难度：⭐⭐ 初级 | 建议学习时间：第5-6周（约10小时）

## 学习目标

- 理解 Zero-shot、One-shot、Few-shot 的区别及适用场景
- 掌握示例选择和排列策略
- 能用 Few-shot 显著提升分类、提取、转换任务的准确率
- 理解 In-context Learning 的工作机制

---

## 1. 三种模式的核心差别

### Zero-shot（零样本）
直接告诉模型要做什么，不提供任何示例：
```
将以下文本的情感分类为：正面/负面/中性

文本："这家餐厅的服务态度太差了！"
分类：
```

### One-shot（单样本）
提供一个示例：
```
示例：
文本："这家餐厅食物超好吃！"
分类：正面

现在分类：
文本："这家餐厅的服务态度太差了！"
分类：
```

### Few-shot（少样本）
提供多个示例，覆盖各种情况：
```
示例1：文本："食物超好吃！" → 分类：正面
示例2：文本："等了1小时还没上菜。" → 分类：负面
示例3：文本："这家店在商场三楼。" → 分类：中性

现在分类：
文本："这家餐厅的服务态度太差了！"
分类：
```

**为什么 Few-shot 更好？**
- 示例消除了格式歧义（输出格式完全确定）
- 示例帮助模型理解分类标准（哪类情况算"负面"？）
- 减少模型自由发挥的空间，输出更稳定

---

## 2. 示例数量的选择

| 模式 | 示例数量 | 适用场景 |
|------|---------|---------|
| Zero-shot | 0 | 通用任务、简单翻译、基础问答 |
| One-shot | 1 | 格式示范、风格迁移 |
| Few-shot | 2-8 | 分类任务、格式转换、特定风格生成 |
| Many-shot | 8+ | 复杂分类、专业领域任务（注意Token成本）|

**经验法则**：
- 任务越复杂 → 需要更多示例
- 示例质量 > 示例数量（3个好示例 > 10个平庸示例）
- 8个示例通常是实用上限

---

## 3. 示例选择策略

### 多样性原则
示例应覆盖任务的各种典型情况：

```
差示例组（都是正面情感，模型只学到"都输出正面"）：
示例1: "产品很好" → 正面
示例2: "质量不错" → 正面
示例3: "非常满意" → 正面

好示例组（覆盖各类情况）：
示例1: "产品很好" → 正面
示例2: "等待时间太长" → 负面
示例3: "包装一般，功能还行" → 中性
示例4: "外观精美但价格贵" → mixed
```

### 边界案例
在示例中包含容易混淆的情况：
```
示例（边界情况）：
文本："包装很漂亮，就是产品本身一般般。"
分类：mixed   ← 告知模型如何处理矛盾情感
```

### 示例顺序的影响
研究表明，**最后一个示例对模型影响最大**（近因效应）。

建议：将最具代表性的示例放在最后，或保持多样化排列。

---

## 4. In-context Learning 的工作机制

Few-shot 学习的本质是**模式识别和格式迁移**，不是真正的"学习"：

1. 模型看到示例，识别出 输入→输出 的模式
2. 将这个模式应用到新的输入上
3. 示例中的格式、风格、分类标准都会被"复制"到输出

**这意味着**：
- 示例中的错误会被模型复制！
- 示例的格式直接决定输出格式
- 示例中隐含的假设会影响分类标准

## 5. Few-shot in CoT（结合思维链）

示例中不只展示答案，还展示推理过程：

```
示例：
文本："这款手机拍照效果惊艳，但电池只有4小时续航，不适合外出使用。"
分析：拍照效果（正面）、续航短（负面）、不适合外出（负面），
     负面因素影响更核心的使用场景
分类：负面

现在分析：
文本："界面设计很漂亮，但每次开机要等30秒，用习惯了倒也还好。"
分析：
分类：
```

这是 Few-shot + CoT 的组合，适合需要解释分类原因的场景。

## 6. 常见应用场景

| 场景 | 所需示例数 | 特点 |
|------|---------|------|
| 情感分类 | 3-5个（覆盖各类别）| 标签需要清晰定义 |
| 格式转换 | 1-3个（示范格式）| 重在展示输入输出格式 |
| 风格迁移 | 2-5个（风格样本）| 示例质量决定输出风格 |
| 数据提取 | 2-3个（带字段说明）| 结合JSON schema效果最佳 |
| 代码翻译 | 1-2个（跨语言示例）| 示例覆盖常见语言特性 |

## 本模块总结

| 关键点 | 核心认知 |
|--------|---------|
| Few-shot vs Zero-shot | 示例消除歧义，提升一致性 |
| 示例数量 | 2-8个最佳，质量>数量 |
| 示例选择 | 多样性 + 边界案例 + 正确标注 |
| 示例顺序 | 最后一个影响最大（近因效应）|
| 工作机制 | 模式识别和格式迁移，不是真正学习 |
