# 模块03 练习题

## 练习 1：设计 Few-shot 分类器（难度：⭐⭐）

为以下任务设计一个 Few-shot 提示词，包含3-5个示例：

**任务**：将用户的问题分类为以下类型：
- `产品咨询`：询问产品功能、价格、规格
- `售后服务`：投诉、退款、维修
- `账户问题`：登录、注册、密码
- `其他`：不属于以上类别

测试用例：
1. "我的手机屏幕碎了，怎么修？"
2. "这款耳机支持蓝牙5.0吗？"
3. "我忘记密码了"
4. "你们公司是哪年成立的？"

<details>
<summary>参考答案</summary>

```
用户问题分类示例：

"这个产品有没有红色款？" → 产品咨询
"我要投诉，等了2周货还没到！" → 售后服务
"我的账号被锁定了怎么解锁" → 账户问题
"你们营业时间是几点到几点？" → 其他

现在分类：
"{用户问题}"
分类：
```

测试答案：
1. 售后服务（维修）
2. 产品咨询
3. 账户问题
4. 其他
</details>

## 练习 2：示例质量优化（难度：⭐⭐）

以下 Few-shot 提示词存在问题，找出并修正：

```
示例1：输入："苹果" → 输出：水果
示例2：输入："苹果公司" → 输出：科技公司
示例3：输入："Apple" → 输出：水果或科技公司

现在分类：
输入："Apple Watch"
输出：
```

问题在哪里？如何改进？

<details>
<summary>参考答案</summary>

**问题**：
1. 示例3的输出"水果或科技公司"包含两个类别，格式不一致
2. 示例未明确说明分类依据（上下文？字面意思？）
3. 没有处理"同名歧义"的示例

**修正版本**：
```
根据输入的具体语境，将其归类为：科技产品/科技公司/水果/其他

示例1：输入："苹果（水果）" → 输出：水果（说明：括号里是上下文提示）
示例2：输入："苹果公司发布了新产品" → 输出：科技公司
示例3：输入："Apple Watch的心率监测功能" → 输出：科技产品

现在分类：
输入："Apple Watch"
输出：
```
（注意：脱离上下文时应标注不确定性）
</details>

## 练习 3：Few-shot 示例数量实验（难度：⭐⭐）

对同一任务（情感分析），分别测试 0/1/3/5 个示例的准确率：

**测试集（10条，正确答案已知）**：
1. "物流很快，包装完好，非常满意" → 正面
2. "质量太差了，买了就坏" → 负面
3. "一般般，没什么特别的" → 中性
4. "客服态度超好，问题都解决了" → 正面
5. "等了3周才到，而且少发了一件" → 负面
6. "按照描述，正常" → 中性
7. "远超预期，强烈推荐！" → 正面
8. "退货很麻烦，押金也没退" → 负面
9. "还可以吧，价格合适" → 中性
10. "一点都不好用，设计很差" → 负面

记录不同示例数量下的正确率，填写表格：

| 示例数量 | 正确数（/10） | 准确率 | 观察到的问题 |
|---------|------------|------|------------|
| 0（Zero-shot） | __ | __%  | |
| 1（One-shot）  | __ | __%  | |
| 3（Few-shot）  | __ | __%  | |
| 5（Few-shot）  | __ | __%  | |

**思考问题**：准确率是否随示例数量单调上升？在哪个数量出现收益递减？

## 练习 4：Few-shot + CoT 设计（难度：⭐⭐⭐）

设计一个 Few-shot + CoT 的提示词，用于判断商业决策的风险等级（高/中/低）。

要求：
- 至少3个示例，每个包含推理过程
- 覆盖高/中/低三个风险等级
- 测试：用一个新的商业决策场景验证效果

## 综合挑战（难度：⭐⭐⭐）

**场景**：你有1000条用户评论需要标注情感（正/负/中），但只能人工标注20条作为示例。

1. 如何选择这20条示例？（考虑多样性、边界案例）
2. 如何设计提示词格式最大化一致性？
3. 如何验证标注质量？
