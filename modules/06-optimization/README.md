# 模块 06：提示词系统性优化

> 难度：⭐⭐⭐⭐ 中高级 | 建议学习时间：第10-11周（约12小时）

## 学习目标

- 建立提示词评估的指标体系
- 掌握 A/B 测试方法论
- 能系统分析提示词失败的根本原因
- 学会在质量、成本、速度之间做权衡

---

## 1. 优化的本质：科学实验

提示词优化不是"试到好为止"，而是一个**科学实验流程**：

```
定义问题 → 建立假设 → 设计实验 → 收集数据 → 分析结果 → 迭代改进
     ↑___________________________________________________|
```

**关键心态转变**：
- 不是"这个提示词不好用" → 而是"在哪些情况下失败？为什么？"
- 不是"多加几个要求" → 而是"哪个要素造成了问题？"

---

## 2. 评估指标体系

### 核心四维度

| 维度 | 评估内容 | 评分方法 |
|------|---------|---------|
| 准确性 | 内容是否正确完整 | 人工评分 1-5 / 与标准答案对比 |
| 格式合规 | 是否按要求输出 | 自动验证（正则/JSON解析）|
| 一致性 | 多次运行是否稳定 | 标准差 / 语义相似度 |
| 成本效率 | Token消耗 vs 输出质量 | Token数 / 质量分数 |

### 评分卡模板

```
任务：[描述]    提示词版本：v[X.X]    测试日期：[日期]
---
测试样本量：[N]次
准确性平均分：[1-5]
格式合规率：[X%]
一致性：[高/中/低]（5次运行标准差）
平均Token消耗：输入[X] / 输出[X]
---
失败案例数：[N]
失败类型分布：[类型1: N次] [类型2: N次]
---
综合评级：[A/B/C/D]
```

## 3. A/B 测试方法

**核心原则：每次只改变一个变量。**

```
❌ 错误方式（同时改变多个变量）：
版本A：你是专家，分析这段文字，用中文，列出要点，简洁
版本B：作为资深分析师，请用结构化格式分析内容，包含摘要和详情
→ 无法判断哪个变化起了作用

✅ 正确方式（单变量控制）：
测试变量：是否有角色定义
版本A（无角色）：分析这段文字，列出要点
版本B（有角色）：作为资深分析师，分析这段文字，列出要点
→ 其他条件完全相同，可以归因
```

**测试流程**：
1. 准备测试集（20-50个有代表性的输入样本）
2. 提前确定评估标准（不能事后调整）
3. 两个版本各运行相同次数
4. 盲评（评分时不知道哪个是A哪个是B）
5. 统计分析（平均分 + 失败率 + 极端case）

## 4. 失败模式诊断

### 指令不清晰
**症状**：方向正确但细节不符合预期
**修复**：增加具体示例，量化约束条件

### 格式漂移
**症状**：有时遵循格式，有时不遵循
**修复**：在格式要求后添加 `"严格按照以上格式，不要偏离"` + 提供格式示例

### 幻觉（Hallucination）
**症状**：模型生成了不存在的信息
**修复**：添加 `"只基于我提供的信息回答，不确定的内容标注[需核实]"`

### 角色出戏
**症状**：角色设定在某些情况下失效
**修复**：找出触发出戏的输入模式，在角色定义中明确处理

### 指令冲突
**症状**：输出在两种行为间摇摆
**修复**：明确指令优先级：`"当X与Y冲突时，优先执行X"`

---

## 5. 迭代优化策略

**最小改动原则**：每次只修改一处，避免"大扫除式"重写（会丢失已有效果）。

**版本控制**：
```
v1.0 - 初始版本 - 准确率60%
v1.1 - 添加角色定义 - 准确率72%（+12%）
v1.2 - 增加输出格式 - 准确率75%（+3%）
v1.3 - 添加边界处理 - 准确率75%，失败率15%→5%（关键改进）
```

**回归测试**：每次修改后，用原有测试集重新测试，确保改进不会破坏原有效果。

---

## 6. 成本优化

| 优化方法 | 节省Token | 风险 |
|---------|---------|------|
| 删除冗余解释 | 10-30% | 低 |
| 简化示例 | 20-40% | 中 |
| 压缩背景信息 | 15-25% | 中 |
| 模型降级（复杂→简单任务）| 视任务而定 | 需测试质量 |
