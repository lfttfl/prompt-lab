## 1. 为什么需要思维链？

**没有 CoT 的问题**：

```
问：一个班25个学生，40%是女生，女生中一半喜欢数学。喜欢数学的女生有几人？
直接输出：5人（可能跳步，直接猜测）
```

**加入 CoT 后**：
```
问：（同上）请一步一步推理，写出每个步骤。

模型输出：
步骤1：班级总人数 = 25人
步骤2：女生数量 = 25 × 40% = 10人
步骤3：喜欢数学的女生 = 10 × 50% = 5人
答案：5人  ← 相同答案，但过程可验证
```

**核心原理**：让模型"外化"推理过程，迫使分步处理，减少跳步错误。研究表明，CoT 在数学推理任务上可将准确率提升 40-50%。

## 2. Standard CoT（标准思维链）

在 Few-shot 示例中，同时展示推理步骤：

```
示例：
问题：工厂每小时产45个零件，8小时/天，6天/周。2000个零件需几周？
推理：
- 每天产量：45 × 8 = 360个
- 每周产量：360 × 6 = 2160个
- 所需周数：2000 ÷ 2160 ≈ 0.93周 → 需要1周（不足1周时取整为1）
答案：1周
```

## 3. Zero-shot CoT

无需示例，只需添加触发语句：

- 中文：`"请一步一步思考。"` 或 `"让我们逐步分析。"`
- 英文：`"Let's think step by step."`

```
问题：水箱有200升水，每分钟漏出3升，同时注入1升新水。多少分钟后水箱变空？
请一步一步思考。
```

**为什么有效**：训练数据中，"step by step" 类语句后通常跟着高质量推理，触发词激活了相关模式。

**局限**：推理步骤可能不够严谨，复杂问题仍不如 Standard CoT 稳定。

## 4. Self-consistency（自洽性采样）

同一问题多次运行，用多数投票选最终答案：

```python
# 概念实现
answers = []
for _ in range(5):  # 运行5次，temperature=0.7
    response = llm(question + "\n请一步一步思考。", temperature=0.7)
    answers.append(extract_final_answer(response))

# 多数投票
final_answer = most_common(answers)
```

**效果**：论文显示在数学题上准确率提升约20%。
**代价**：5倍 API 调用成本。
**适用场景**：高准确率需求、成本可接受的关键任务。

## 5. Tree of Thoughts（思维树）

不是线性推理链，而是树状探索多个路径：

```
提示词实现：
这道题需要多步推理：[问题]

请按以下步骤：
1. 列出3种可能的解题思路（只列思路，不深入）
2. 评估每种思路的可行性（1-5分），说明理由
3. 选择得分最高的思路，展开完整推理
4. 得出答案
```

---

## 6. Least-to-Most Prompting（从简到繁）

将复杂问题分解为子问题，逐步解决：

```
第一步（分解）：
提示词："要解决[复杂问题]，我需要先解决哪些更简单的子问题？列出。"

第二步（逐步求解）：
提示词："已知[子问题1的答案]，现在解决[子问题2]..."
```

---

## 7. 何时使用 CoT？

| 任务类型 | 推荐方式 | 说明 |
|---------|---------|------|
| 简单问答 | 直接提问（不需要CoT）| CoT会增加延迟和成本 |
| 数学计算 | Zero-shot CoT 或 Standard CoT | 明确要求"逐步计算" |
| 多步逻辑 | Standard CoT | 需要展示清晰推理链 |
| 高准确率需求 | Self-consistency | 代价是多倍API调用 |
| 策略规划 | Tree of Thoughts | 需要探索多种可能性 |

---

## 本模块总结

| 技术 | 代价 | 收益 | 适用场景 |
|------|------|------|---------|
| Standard CoT | 更多Token输入 | 稳定推理格式 | 有标准推理模式的任务 |
| Zero-shot CoT | 极低 | 推理准确率大幅提升 | 快速提升推理质量 |
| Self-consistency | 3-5倍API调用 | 准确率显著提升 | 高准确率关键任务 |
| Tree of Thoughts | 高 | 复杂问题最优解 | 策略规划、创意探索 |
