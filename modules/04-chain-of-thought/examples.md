# 模块04 实践示例

## 示例 1：Zero-shot CoT 触发词效果

**任务**：多步数学应用题

**无 CoT**：
```
提示词：小明有24块糖，分给3个朋友，每人先拿了2块，
       然后小明又买了12块，现在总共有多少块？
输出：30块（可能跳步出错）
```

**加 Zero-shot CoT**：
```
提示词：（同上）请一步一步思考。
输出：
步骤1：初始糖数 = 24块
步骤2：3个朋友各拿2块 = 共拿走 3×2=6块
步骤3：剩余：24-6=18块
步骤4：又买了12块：18+12=30块
答案：30块
```

两个答案相同，但 CoT 版本的过程可以验证，更可靠。

## 示例 2：Standard CoT 解决逻辑推理

**任务**：复杂逻辑题

**提示词（含CoT示例）**：
```
示例：
问题：A比B高，B比C高，D比A矮，谁最高？
推理：
- A > B（A比B高）
- B > C（B比C高）
- 所以 A > B > C
- D < A（D比A矮）
- 因此从高到低：A > B > C，D在A之下但位置不确定
- 最高的是 A
答案：A最高

现在解决：
问题：红队比蓝队快，绿队比红队慢，黄队比绿队快但比蓝队慢，谁最快？
推理：
```

模型会按照示例中的步骤格式，一步步推理，准确给出"蓝队最快"。

## 示例 3：Self-consistency 提升关键任务准确率

**场景**：判断一封邮件是否是钓鱼邮件（安全关键任务）

```python
# 运行5次，各自独立推理
results = []
for i in range(5):
    prompt = f"""
    分析以下邮件是否是钓鱼邮件，请逐步检查以下特征：
    1. 发件人域名是否可疑
    2. 是否有紧迫性语言
    3. 是否要求提供敏感信息
    4. 链接是否与声称的来源匹配
    
    邮件内容：[邮件]
    
    逐步分析，最后给出：钓鱼邮件/正常邮件
    """
    results.append(llm(prompt, temperature=0.3))

# 5次结果：[钓鱼, 钓鱼, 正常, 钓鱼, 钓鱼] → 投票结果：钓鱼（4:1）
```

## 示例 4：Tree of Thoughts 用于商业决策

**场景**：公司是否应该进入新市场

```
这是一个重要的商业决策，需要多角度分析。

问题：我们的B2B SaaS产品是否应该进入东南亚市场？

请按以下步骤：

第一步：列出3个可能的分析框架（每个框架只用一句话描述）
第二步：评估每个框架的适用性（1-5分）
第三步：用得分最高的框架展开完整分析
第四步：给出建议和下一步行动
```

输出会系统地探索不同视角（市场规模分析 vs 竞争格局分析 vs 资源能力分析），然后深入最相关的一个。

## 示例 5：CoT 用于代码调试

**场景**：让模型系统地调试一段有 bug 的代码

```
以下Python代码有bug，请一步一步排查：

代码：
def calculate_average(numbers):
    total = 0
    for num in numbers:
        total += num
    return total / len(numbers)

result = calculate_average([])
print(result)

调试步骤：
1. 分析代码逻辑是否正确（在有数据时）
2. 分析边界情况（空列表时会发生什么）
3. 找出具体的错误行和错误原因
4. 提供修复方案
```
