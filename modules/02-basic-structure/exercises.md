## 练习 1：诊断提示词问题（难度：⭐）

找出以下提示词的问题，并说明如何改进：

1. `"帮我写点东西关于可持续发展"`
2. `"分析一下市场"`
3. `"做一个总结，要好"`
4. `"给我一些建议"`

<details>
<summary>参考答案</summary>

1. 问题：没有角色、没有具体任务、没有受众、没有格式。改进：加入"你是环保领域记者，为高中生写一篇关于可持续发展的500字科普文章，用3个具体案例说明..."

2. 问题："市场"太模糊，没有背景、没有格式。改进：明确是哪个市场、从什么角度、输出什么格式。

3. 问题："好"是主观词汇，没有量化标准。改进：明确总结的结构和长度要求。

4. 问题：完全没有指定主题、背景、数量、格式。改进：加入具体情境、受众、建议数量和格式。
</details>

## 练习 2：重写一个提示词（难度：⭐⭐）

将以下提示词用六要素框架完整重写：

**原版**：
```
帮我写一份会议纪要
```

**要求**：假设这是一个产品团队的需求评审会议，你是一个助理，需要输出正式的会议纪要文档。

<details>
<summary>参考答案</summary>

```
【角色】你是一名专业的行政助理，擅长整理结构化会议纪要。

【任务】根据以下会议内容，整理一份正式的需求评审会议纪要。

【背景】这是一家互联网公司产品团队的周度需求评审会议。
参与人员包括产品经理、设计师、前端和后端工程师。

【格式】
- 文档标题：[会议主题] 会议纪要
- 会议基本信息（时间/地点/参与人/主持人）
- 议题一览（编号列表）
- 每个议题的讨论结果和决议（要点格式）
- 待办事项（责任人 + 截止日期表格）
- 下次会议时间

【约束】
- 使用客观、正式的语气，不加个人判断
- 待办事项必须有明确的责任人
- 结论和讨论过程要区分清楚

【输出】直接输出纪要文档，不需要前言。

---
会议内容：[粘贴会议记录或录音转写]
```
</details>

## 练习 3：JSON 格式输出（难度：⭐⭐）

设计一个提示词，输出包含：姓名、年龄、学历、技能列表、工作经历、期望薪资。

## 练习 5：约束词实验（难度：⭐⭐⭐）

设计一个"最小有效提示词"：用不超过50个字，完成以下任务——

**任务**：让模型将任意输入的中文句子翻译成英文，并给出3个不同风格的版本（正式/口语/文学）。

**挑战**：在50字限制内，如何同时传达：任务、格式、风格要求？

<details>
<summary>参考答案（参考，不唯一）</summary>

```
将输入的中文翻译为3个英文版本，格式：
正式版：[翻译]
口语版：[翻译]
文学版：[翻译]

输入：{text}
```
（约35字，简洁有效）
</details>

## 综合挑战（难度：⭐⭐⭐）

**任务**：为一个"代码审查助手"设计完整的系统提示词。

要求：
- 目标用户：有2-5年经验的开发者
- 审查维度：代码质量 + 安全漏洞 + 性能优化
- 输出格式：结构化报告（必须有严重性分级）
- 限制：只审查，不直接修改代码
- 语言：用中文输出，代码示例用原语言

设计你的系统提示词，然后用一段你写过的代码测试它。
