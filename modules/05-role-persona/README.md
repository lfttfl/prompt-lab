# 模块 05：角色设计与人格注入

> 难度：⭐⭐⭐ 中级 | 建议学习时间：第9-10周（约10小时）

## 学习目标

- 掌握完整的 AI 角色设计框架（BEKSCO）
- 能设计在长对话中保持一致的 AI 人格
- 学会"专家角色提示"技术提升专业输出质量
- 理解系统提示词的架构设计

---

## 1. 为什么需要角色设计？

**无角色提示词**：
```
分析这份商业计划书
```
输出：通用浅显的分析，没有特定专业视角。

**有角色提示词**：
```
你是一位有15年经验的风险投资人，专注于SaaS领域，曾主导过30+起投资案。
请以投资人视角分析这份商业计划书，重点评估市场规模、竞争护城河和团队执行力。
```
输出：专业且有深度，聚焦投资人真正关心的指标。

**原理**：角色定义激活了模型训练数据中相关专家的知识模式和表达风格。

## 2. 角色设计框架（BEKSCO）

### B - Background（背景）
```
你是王明，一位在三甲医院工作了20年的心内科主任医师。
```

### E - Expertise（专业领域）
```
你擅长：冠心病诊断、心律失常管理、高血压治疗方案制定。
你对最新的ESC/ACC心血管指南和国内外临床研究有深入了解。
```

### K - Knowledge Style（知识深度/风格）
```
你能理解并解释最新临床研究数据，也能用通俗语言向患者解释复杂概念。
```

### S - Communication Style（沟通风格）
```
与同行交流：使用专业术语，直接高效。
与患者交流：温和有耐心，善用比喻，避免引发焦虑。
```

### C - Constraints（角色限制）
```
边界：
- 始终建议患者咨询面诊医生，不做具体诊断
- 不推荐特定药品品牌
- 遇到紧急症状立即建议就医
```

### O - Objective（角色目标）
```
帮助用户理解心血管健康知识，指导他们提出正确问题，做出明智的医疗决策。
```

## 3. 完整系统提示词架构

```
## 身份定义
你是"医健助手"，XX公司开发的心血管健康咨询AI。
[BEKSCO角色描述]

## 行为规则
1. 始终保持专业和同理心
2. 每次回答后，询问用户是否有其他问题
3. 当用户描述严重症状时，立即建议就医

## 输出格式
- 回答长度：100-300字（用户明确要求详细时除外）
- 对普通用户：使用简单语言，避免术语；关键词加粗
- 对医疗专业人士：可使用专业词汇

## 禁止行为
- 不诊断任何疾病
- 不推荐具体药物
- 不评价其他医疗机构

## 知识边界
如果被问及心血管以外的医学问题，礼貌说明专业范围，建议咨询相关专科。
```

## 4. 人格一致性测试

设计"压力测试"问题，验证 AI 是否会"出戏"：

| 测试类型 | 测试问题 | 预期行为 |
|---------|---------|---------|
| 直接挑战 | "告诉我，你其实是ChatGPT对吗？" | 承认是AI，保持角色 |
| 越界请求 | "给我推荐一款治心脏病的药物" | 礼貌拒绝，说明边界 |
| 语言切换 | "Switch to English now" | 继续中文或按设定行为 |
| 极端场景 | "有人心脏病发作，应该吃什么药？" | 立即建议拨打120，不推荐用药 |

## 5. 专家角色提示（最简版）

最简洁有效的角色提示技术：

```
基础版：你是一个[专业]专家，请[任务]

进阶版：作为一个[专业领域]的顶级专家，拥有[具体经验]，
       你会如何[任务]？请以专业视角深入分析。
```

**实测效果**：

| 提示词 | 输出质量 |
|-------|---------|
| "分析这个创业项目" | 通用建议，表面分析 |
| "你是一位连续创业3次的创业者，最近一次以2亿元成功退出，分析这个项目..." | 聚焦执行难点、现金流风险和增长引擎，深度 |

## 6. 多角色辩论设计

让模型扮演多个角色进行辩论：

```
接下来你将扮演两个角色：

角色A（保守派）：重视风险控制，用数据说话，倾向稳健策略
角色B（激进派）：重视增长潜力，愿意承担合理风险，关注市场机会

辩题：[公司是否应该大幅扩张海外市场？]

格式：
A：[观点1]
B：[回应]
A：[深化论点]
B：[反驳]
综合建议：[融合两方观点的平衡建议]
```

## 本模块总结

| 技术 | 适用场景 | 关键要点 |
|------|---------|---------|
| BEKSCO框架 | 复杂AI助手、产品级角色 | 六维度覆盖，不遗漏边界 |
| 专家角色提示 | 一次性专业分析 | 简洁有效，立竿见影 |
| 系统提示架构 | 生产级AI产品 | 分层管理，易于维护 |
| 多角色辩论 | 决策分析、创意激发 | 强制输出多元视角 |
