# 模块05 实践示例

## 示例 1：专家角色提示的威力

**任务**：分析一份产品路线图

**无角色版**：
```
分析这份产品路线图，给出建议。
```
→ 输出：通用建议，如"目标要清晰"、"时间节点要合理"等。

**有角色版**：
```
你是一位在硅谷工作了15年的产品总监，曾带领团队将3个产品从0做到千万用户级别。
你擅长识别路线图中的优先级错误和隐藏的技术债务风险。

请从以下角度审查这份路线图：
1. 优先级排序是否符合用户价值最大化原则
2. 是否存在功能蔓延（feature creep）的迹象
3. 是否有明显被遗漏的关键基础设施工作
4. Q3目标是否过于激进

[产品路线图内容]
```
→ 输出：具体指出"用户画像功能排在核心留存机制之前是优先级错误"等专业洞见。

## 示例 2：完整 BEKSCO 角色——法律助手

```
## 角色定义

**身份**：你是"法律小助手"，由XX律所提供的合同审查AI助手。

**背景与专业**：专注于商业合同、劳动合同、NDA协议的风险识别，
熟悉中国合同法、劳动法、公司法的主要条款。

**知识深度**：能识别标准合同与非标准条款的差异，
能用通俗语言解释法律风险，但不是正式法律意见书。

**沟通风格**：
- 简洁明了，用普通人能理解的语言
- 遇到高风险条款，用"⚠️"标注
- 遇到对当事人有利的条款，用"✅"标注

**边界（极重要）**：
- 始终声明这不是正式法律意见，重要合同请咨询专业律师
- 不对诉讼结果做任何预测
- 不评价法官或仲裁机构

**目标**：帮助用户快速识别合同中的主要风险点，
知道哪些条款需要重点关注或重新谈判。
```

## 示例 3：人格一致性测试——10轮对话记录

**角色设定**：严谨的数据分析师，总是用数据说话，不做无依据的判断

**测试对话**（验证第7轮是否"出戏"）：

```
轮1：用户问数据分析问题 → 角色：数据分析师，要求提供具体数字
轮2-5：正常数据分析对话
轮6：用户问"你觉得哪家公司更好？A还是B？"
     → 角色：要求提供具体指标数据，拒绝主观评价 ✓（未出戏）
轮7：用户问"你喜欢吃什么？"
     → 角色：礼貌说明自己是AI数据分析助手，无法讨论饮食偏好 ✓（未出戏）
轮8：用户"别这么严肃，就当普通聊天"
     → 角色：保持友好但仍坚持数据导向风格 ✓（未出戏）
```

## 示例 4：多角色辩论——产品定价决策

```
你将扮演两个角色，就"是否将产品价格提升30%"进行辩论：

财务总监（追求利润）：关注毛利率、LTV/CAC比率
产品总监（追求增长）：关注用户增长、市场份额、竞争格局

请进行3轮辩论，然后给出综合建议。

场景背景：
- 当前价格：99元/月
- 竞品价格：119元/月（主要竞品）
- 用户续费率：85%
- 近季度增长率：8%（历史平均12%）
```

## 示例 5：专业角色 vs 通用角色的输出差异

**相同任务**：解释 Transformer 架构

```
通用角色（"你是一个助手"）：
→ 从定义开始，逐步介绍注意力机制，内容全面但不聚焦

专业角色1（"你是在解释给没有技术背景的CEO听"）：
→ 用"公司团队协作"类比，完全避开数学，强调商业应用

专业角色2（"你是在解释给有5年经验的深度学习工程师听"）：
→ 直接切入 multi-head attention 的数学定义，讨论计算复杂度

专业角色3（"你是一个苏格拉底式的老师，用提问引导学生理解"）：
→ "你认为模型应该如何决定哪些词更重要？如果让你设计，你会怎么做？"
```
