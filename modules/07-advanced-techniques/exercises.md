# 模块07 练习题

## 练习 1：RAG 提示词设计（难度：⭐⭐⭐）

为以下场景设计完整的 RAG 提示词链（包含查询改写 + 上下文注入）：

**场景**：HR 部门的政策问答机器人，知识库包含员工手册、假期政策、薪酬规定。

**测试问题**：
1. "我能请几天年假？"
2. "生育假怎么申请？"
3. "绩效奖金什么时候发？"

设计：
- 查询改写提示词（如何把口语化问题改写成精确检索词）
- 上下文注入提示词（如何控制引用格式和不确定性说明）

---

## 练习 2：工具描述设计（难度：⭐⭐）

为以下工具设计高质量的 Function Calling 描述（特别注意 `description` 字段）：

1. 一个查询天气的工具（参数：城市、日期）
2. 一个发送邮件的工具（参数：收件人、主题、正文）
3. 一个数据库查询工具（参数：SQL 语句）

对于工具3，思考：应该如何在描述中限制 SQL 的危险操作？

---

## 练习 3：ReAct Agent 提示词设计（难度：⭐⭐⭐）

设计一个能处理以下任务的 ReAct Agent 提示词：

**工具**：calculator（计算）、search（搜索）、get_current_date（获取今天日期）

**任务**：回答"距离2026年春节还有多少天？"

写出：
1. 完整的系统提示词（包含工具描述和ReAct格式）
2. 预期的模型输出过程（思考-行动-观察-答案）

---

## 练习 4：提示词链设计（难度：⭐⭐⭐）

设计一个3步提示词链，用于处理用户投诉：

**输入**：一条用户投诉文本
**输出**：标准化的客服工单

步骤设计：
1. 步骤1：情绪识别 + 问题分类
2. 步骤2：优先级判断 + 处理建议
3. 步骤3：生成标准化工单（JSON格式）

为每个步骤写出具体的提示词。

---

## 综合挑战（难度：⭐⭐⭐⭐⭐）

设计一个完整的"智能简历筛选"系统提示词架构：

1. **查询理解层**：将 HR 的要求（如"需要5年Python经验，有AI项目经历"）转换为结构化筛选标准
2. **匹配评分层**：将简历内容与筛选标准逐条对比，给出分数
3. **结果汇总层**：生成包含评分、亮点、疑虑点的简历评估报告

要求：支持批量处理多份简历，输出统一格式便于比较。
