# 基础结构模板（RTCFCE）

> 适用场景：日常通用任务、内容生成、信息处理
> 核心技术：六要素结构框架

---

## 模板（系统提示词版本）

```
【角色 Role】
你是{角色描述，如：一位有X年经验的Y领域专家}。

【任务 Task】
{动词开头的具体任务，如：分析/撰写/提取/转换}以下{内容类型}。

【背景 Context】
目标受众：{谁会使用这个输出}
使用场景：{在什么情况下使用}
特殊背景：{其他需要了解的背景信息，无则删除此行}

【格式 Format】
输出结构：{描述输出的结构，如：标题+3个要点+总结}
长度要求：{字数范围，如：200-300字}
格式类型：{纯文本/Markdown/JSON/表格，选一个}

【约束 Constraints】
- 不要{不想要的行为1}
- 不要{不想要的行为2}
- 只使用{限制来源}中的信息（如适用）

【示例 Examples（可选）】
示例输入：{示例}
示例输出：{期望的输出格式和风格}

【直接开始，不要前言】
```

## 精简单提示词版本（不分系统/用户）

```
作为{角色}，请{任务}。

背景：{关键背景信息}
要求：
- 格式：{格式要求}
- 长度：{长度要求}
- 约束：{主要限制}

输入内容：
{user_input}
```

---

## 使用示例

```
【角色】你是一位有8年经验的产品经理，擅长将技术需求转化为用户友好的说明文档。

【任务】将以下技术变更说明改写为面向普通用户的版本。

【背景】
目标受众：非技术背景的企业用户（财务、HR、运营人员）
使用场景：产品更新公告推送给用户

【格式】
结构：变更概述（1句话）+ 对我有什么影响（2-3条）+ 需要我做什么（步骤列表）
长度：150字以内
格式：纯文本，重要信息加粗

【约束】
- 不使用任何技术术语（API、数据库、前端等）
- 不夸大变更的重要性
- 如果变更对用户无感知，直接说明无需操作

技术变更说明：
{tech_change_description}
```

---

## 适配建议

| 任务类型 | 最重要的要素 | 可以简化的要素 |
|---------|----------|------------|
| 内容生成 | Role + Format | Context（如果很清晰）|
| 信息提取 | Task + Format + Constraints | Role |
| 分析报告 | Role + Task + Context + Format | Examples |
| 格式转换 | Task + Format + Examples | Role + Context |
