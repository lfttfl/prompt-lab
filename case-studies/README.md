## 每个案例的结构

每个案例都包含：
1. **业务背景**：为什么需要这个AI系统
2. **需求分析**：具体的功能和质量要求
3. **设计过程**：提示词从v1到最终版的演进
4. **失败案例**：遇到了什么问题，如何解决
5. **最终提示词**：可直接使用的完整版本
6. **测试验证**：如何验证效果

## 建议使用方式

1. 先自己尝试设计，再对照案例
2. 重点关注"失败案例"部分，了解踩坑经历
3. 参考"设计过程"理解为什么这样迭代
4. 复制"最终提示词"到自己的提示词库中
