# 案例02：结构化数据提取系统

## 业务背景

一家招聘平台每天收到数千份非结构化简历（Word/PDF转文本），需要自动提取候选人信息并录入数据库。人工录入成本高且出错率约8%，亟需自动化解决方案。

## 需求分析

**输入**：非结构化简历文本（格式多样，无统一模板）
**输出**：JSON格式的结构化数据，包含25个字段
**质量要求**：字段提取准确率 >95%；不存在的字段输出null（不能编造）；JSON格式100%有效

---

## 设计过程

### v1.0 —— 失败：大量幻觉

```
从以下简历中提取信息，输出JSON格式：
[简历内容]
```

**问题**：
- 模型会"填充"不存在的信息（如简历未写薪资，模型会猜测一个）
- JSON格式不稳定，有时在JSON外添加说明
- 字段名称不一致

---

### v1.1 —— 改进：指定Schema，但仍有幻觉

```
从以下简历中提取信息，按以下JSON schema输出：
{"name": "", "age": 0, "education": "", ...}
[简历内容]
```

**改进**：字段名称统一了，但模型仍会为空字段填充内容。

---

### v1.2 —— 关键改进：明确空字段处理规则

```
从以下简历文本中提取候选人信息。

【严格规则】
- 只提取简历中明确存在的信息
- 简历中没有提到的字段，必须输出null，绝对不能推断或填充
- 只输出JSON，不输出任何其他内容

【JSON Schema】
{
  "personal": {
    "name": "字符串或null",
    "age": "数字或null",
    "gender": "字符串或null",
    "location": "字符串或null",
    "phone": "字符串或null",
    "email": "字符串或null"
  },
  "education": [{
    "degree": "学历级别",
    "major": "专业",
    "school": "学校名",
    "graduation_year": "数字或null"
  }],
  "work_experience": [{
    "company": "公司名",
    "title": "职位",
    "start_date": "YYYY-MM或null",
    "end_date": "YYYY-MM或'至今'或null",
    "responsibilities": ["职责1", "职责2"]
  }],
  "skills": ["技能1"],
  "languages": [{"language": "语言", "level": "水平"}],
  "salary_expectation": {
    "min": "数字（元/月）或null",
    "max": "数字（元/月）或null"
  },
  "availability": "字符串或null",
  "summary": "简历总体评价（50字以内，基于简历内容）"
}

【简历内容】
{resume_text}
```

---

### v1.3（最终版）—— 加入验证层

在主提取提示词之后，增加一个验证步骤：

```
以下是从简历中提取的JSON数据和原始简历文本。
请验证JSON中每个非null字段，确认在原简历中有对应依据。
如果发现某字段的值在简历中找不到明确依据，将其改为null。

原简历：{resume_text}
提取的JSON：{extracted_json}

输出修正后的JSON（如无需修正，输出原JSON）。
```

---

## 测试验证

**测试集**：100份真实简历（已人工标注正确答案）

| 指标 | v1.0 | v1.2 | v1.3（带验证）|
|------|------|------|------|
| 字段准确率 | 72% | 91% | 97% |
| 幻觉率（填充不存在信息）| 23% | 6% | 1% |
| JSON格式有效率 | 85% | 98% | 99% |
| 处理成本（相对）| 1x | 1x | 1.4x（增加验证步骤）|

---

## 关键设计洞察

1. **"只提取存在的信息"必须明确说明**：不加这条限制，模型会主动"补全"
2. **空字段的处理规则要精确**：`null` 比 `""` 或 `"未知"` 更利于后续数据处理
3. **两步走（提取+验证）** 比单步提取准确率高，成本增加约40%但值得
4. **Few-shot 示例对复杂Schema特别有效**：示例展示如何处理边界情况（同名不同格式等）
